# ROS2 Brain–Cerebellum Agent 架构需求计划（完整版 v0.1）

---

# 一、项目概述

## 1.1 项目名称

**ros2-brain-agent**

副标题：

> A Governable Brain–Cerebellum Agent Architecture for ROS2

---

## 1.2 项目目标

构建一个基于 ROS2 的 Brain–Cerebellum 协同软件架构，支持：

- 对话能力（LLM）
- 语音能力（ASR / TTS）
- 记忆系统（文件系统实现，后续可扩展）
- 工具调用（Tool / Skill）
- 导航与机械臂协同
- 错误恢复与执行语义
- 安全治理与审计
- 可观测事件流

所有 AI 能力通过外部 API 接入，系统专注于：

> 架构抽象 + 执行语义 + 工程治理

---

# 二、总体架构设计

## 2.1 分层架构

```
                ┌────────────────────────┐
                │        Brain           │
                │ Dialog / Agent Core    │
                │ Plan / Memory / Tool   │
                └───────────┬────────────┘
                            │ SkillInvoke
                ┌───────────▼────────────┐
                │      Cerebellum        │
                │   Skill Server Layer   │
                │ Nav2 / MoveIt2 Adapter │
                └───────────┬────────────┘
```

语音链路：

```
ASR  →  Brain

```

# 三、功能需求

---

# 1️⃣ 对话与 Agent 核心能力

## 1.1 会话管理

- 支持多 `session_id`
- 每轮 turn 独立结构化记录
- 所有行为具备完整审计日志

记录内容：

- 用户输入
- LLM 输出
- tool 调用
- skill 执行结果
- 错误与恢复行为

事件发布 Topic：

```
/dialog/events
```

---

## 1.2 LLM 输出规范

LLM 必须输出结构化 JSON：

```json
{
  "assistant_text": "回复文本",
  "plan": [],
  "tool_calls": [],
  "memory_write": []
}
```

要求：

- JSON Schema 校验
- 自动修复重试机制
- 动态注入 tool 描述
- 输出可审计、可回放

---

## 1.3 Agent 执行语义

支持以下执行语义：

- sequence（串行）
- parallel（并行）
- conditional（条件分支）
- retry（重试）
- compensate（补偿动作）

标准执行流程：

```
User Input
   ↓
Memory Query
   ↓
LLM Plan
   ↓
Tool Router
   ↓
Skill Execute
   ↓
Feedback
   ↓
Memory Update
   ↓
Output
```

---

# 2️⃣ 记忆系统（v0.1 文件系统实现）

---

## 2.1 设计原则

- 不依赖外部数据库组件
- 基于本地文件系统
- 可插拔后端（通过 MemoryStore 抽象）
- Append-only 优先
- 可审计、可回放
- 结构清晰便于未来迁移至 SQLite / Redis

---

## 2.2 记忆分层结构

### 2.2.1 短期记忆（Working Memory）

- 最近 N 轮对话
- 用于拼接 LLM 上下文
- 存储形式：`turns.jsonl`（append-only）

---

### 2.2.2 会话摘要（Session Summary）

- 周期性压缩历史对话
- 用于长对话上下文恢复
- 存储形式：`summary.json`

---

### 2.2.3 用户事实（Facts / Profile）

- Key-Value 存储
- 示例：用户名、语言偏好、常用地点
- 存储形式：`facts.json`

---

### 2.2.4 事件日志（Events）

- tool 调用
- skill 执行
- 错误与恢复
- 耗时统计
- 存储形式：`events.jsonl`

---

### 2.2.5 支持设置长期记忆

- 可以通过对话，修改调整记忆内容

## 2.3 文件系统目录结构

```
memory/
  sessions/
    <session_id>/
      turns.jsonl
      events.jsonl
      summary.json
      facts.json
  global/
    user_facts.json
```

---

## 2.4 数据格式示例

### turns.jsonl

```json
{
  "turn_id": 12,
  "ts": "2026-02-17T10:00:00Z",
  "speaker": "user",
  "text": "帮我把杯子拿过来"
}
```

---

### events.jsonl

```json
{
  "event_id": "e-1001",
  "ts": "2026-02-17T10:00:03Z",
  "type": "tool_invoke",
  "session_id": "s1",
  "payload": {
    "tool": "nav2.goto",
    "duration_ms": 5320,
    "ok": true
  }
}
```

---

### summary.json

```json
{
  "version": 3,
  "updated_at": "2026-02-17T10:05:00Z",
  "summary_text": "用户希望机器人去桌子取杯子并送到身边。"
}
```

---

### facts.json

```json
{
  "schema_version": 1,
  "facts": {
    "user_name": "川",
    "preferred_language": "zh"
  }
}
```

---

## 2.5 Memory 抽象接口

定义内部接口：

```
MemoryStore
  - append_turn()
  - append_event()
  - get_recent_turns()
  - get_summary()
  - set_summary()
  - get_facts()
  - upsert_facts()
```

默认实现：

```
FileSystemMemoryStore
```

预留扩展：

```
SQLiteMemoryStore
RedisMemoryStore
```

# 3️⃣ 工具与技能系统

---

## 3.1 三层能力模型

### Primitive（原子能力）

- nav2.goto
- arm.move_to
- perception.detect

---

### Skill（复合能力）

- skill.approach_for_pick
- skill.pick_object
- skill.deliver_object

---

### Task（任务）

由 LLM 生成，由 Brain 调度执行。

---

## 3.2 Skill 接口

```
SkillExecute.action
```

字段：

- goal:
  - skill_name
  - args_json
- feedback:
  - progress
  - message
- result:
  - ok
  - result_json
  - error

---

## 3.3 工具注册表（tools.yaml）

包含：

- tool_name
- type
- JSON schema
- 权限等级（safe / confirm / dangerous）
- 限流策略
- preconditions
- postconditions
- timeout
- retry

---

# 4️⃣ 机械臂 + 导航协同设计

---

## 4.1 协同原则

- Brain 不直接调用 primitive
- Brain 只调用 Skill
- 小脑封装局部恢复与感知闭环

---

## 4.2 World State 节点

维护：

- robot_pose
- localization_ok
- nav_state
- arm_state
- holding_object
- perceived_objects
- safety_state

提供：

```
WorldStateQuery.srv
```

---

## 4.3 错误码体系

示例：

- OBJECT_NOT_FOUND
- NAV_TIMEOUT
- LOCALIZATION_UNSTABLE
- GRASP_FAILED
- SAFETY_ESTOP

统一结构：

- code
- category
- recoverable
- suggested_actions
- context

---

# 5️⃣ 安全与治理

必须支持：

- 工具白名单
- 参数 Schema 校验
- 权限分级控制
- 危险动作确认机制
- 调用限流
- 审计日志
- dry-run 模式

所有 ROS action/service 必须通过：

```
tool_router_node
```

禁止 LLM 直接访问 ROS。
